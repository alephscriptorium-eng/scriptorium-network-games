### Análisis y Desglose de la Entropía (La Cuestión)

Tu idea es construir un **sistema transmedia** que utiliza el conocimiento acumulado de Wikipedia como base para clasificar las diferentes líneas de pensamiento (demarcación, pseudociencia) en múltiples idiomas.

Aquí está la entropía: Estás mezclando la arquitectura del sistema (caching, cadenas de investigación) con la ontología del contenido (lenguajes, conceptos filosóficos). Para ordenarlo, debemos separar los elementos.

#### 1. Desglose de Variables Clave

| Variable | Definición/Problema | Entropía Relevante |
| :--- | :--- | :--- |
| **Sistema Transmedia** | El navegador que cachea las investigaciones y construye *chains*. | ¿Cómo se relaciona el "cacheo" con la "evidencia"? |
| **Lenguajes (N)** | La cantidad de idiomas a analizar. | Necesitas un límite definido. ¿Todos los idiomas o solo los más relevantes? |
| **Categorías** | Demarcación y Pseudociencia. | Estas categorías deben ser operacionales (definibles) en cada idioma. |
| **Evidencia Mínima** | La cantidad de líneas/datos requeridos. | Esta es la variable crucial que depende del umbral de confianza que quieras establecer. |

#### 2. Reducción de Entropía: Propuesta Metodológica

En lugar de buscar una respuesta, vamos a definir las **reglas** del juego para el sistema. Esto te da control y estructura.

**Paso A: Definir el Alcance (Reducir N)**
Antes de intentar abarcar *todos* los idiomas del mundo, debes reducir la variable $N$.

*   **Pregunta clave:** ¿Cuáles son los **N=5 o N=10 lenguajes más relevantes** para tu máquina o sistema transmedia? (Ejemplo: Inglés, Francés, Alemán, Español, Chino). Esto te da un punto de partida manejable.
*   **Acción:** Define el universo lingüístico que tu sistema debe priorizar.

**Paso B: Definir la Evidencia Mínima (Reducir la Cantidad)**
La cantidad de evidencia necesaria depende del nivel de confianza ($C$) que busques.

*   **Estrategia:** Establece un **Umbral de Validación**. Por ejemplo, si quieres afirmar una "línea" sobre demarcación en un idioma específico, ¿cuántas referencias coincidentes (de diferentes fuentes o incluso Wikipedias) deben aparecer para considerarlo válido?
    *   Ejemplo: Para establecer que una línea es de "Pseudociencia", se requieren al menos **3 citas** consistentes entre las entradas de Wikipedia de ese idioma.

**Paso C: Estructurar la Matriz (El Plan)**
Tu tabla debe ser el mapa, no solo un contenedor. Propongo esta estructura para organizar el proceso:

| Idioma ($L_i$) | Categoría ($C_j$: Demarcación/Pseudociencia) | Métrica de Evidencia Requerida ($E_{min}$) | Estado Actual (Cacheado) | Fases Siguientes |
| :--- | :--- | :--- | :--- | :--- |
| Inglés | Pseudociencia | 3 citas coincidentes con fuente primaria. | Pendiente | **Buscar y Validar** |
| Español | Demarcación | 1 referencia clave (si es un concepto central). | Recolectado | **Correlacionar** |
| ... | ... | ... | ... | ... |

## Descomponer la entropía (tres capas)

```
INVENTARIO (N=70)     →  ¿existe entrada en ese idioma?
SUBSTANCIA (N′ << 70) →  ¿hay bytes/secciones para clasificar?
LÍNEA (🟡)            →  ¿qué postura editorial muestra el wikitext?
```

Confundir las tres es la fuente principal de caos.

**Máquina B** (transmedia) no es «N iteraciones = N historiales completos». Es:

1. **Inventario** — Wikidata sitelinks (1–2 calls, sin wikitext)  
2. **Probe** — meta por idioma: bytes + secciones del latest (2×N calls API, sleep 1s)  
3. **Clasificación** — rubrica sobre snapshots 🟢, no sobre memoria del modelo  
4. **Chains** — una investigation chain por cluster o por idioma sustantivo

Solve et Coagula (A) = expediente forense **un evento, un idioma**. Máquina B = **matriz comparativa** con el mismo contrato de caché (`fetch_snapshot.py`, `CACHE_RUNBOOK.md`).
